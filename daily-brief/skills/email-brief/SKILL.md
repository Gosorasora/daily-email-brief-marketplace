# email-brief

## ⚙️ 설정 (사용 전 채우기)
- `<PRIMARY_EMAIL>` : 메인 Gmail 주소
- `<FORWARDED_EMAILS>` : 메인으로 자동전달되는 보조 주소 (없으면 "없음")
  - 외부 메일(예: 네이버 등)은 Gmail POP 수집 종료(2026)로 미유입. 포함하려면 해당 계정→Gmail **자동전달** 설정. 그 전까짃 리포트 하단에 누락 명시왔.
- `~~email` 커넥터(Gmail MCP) 연결 필요.

## 1. 조회
- Gmail MCP `search_threads`: `query: newer_than:1d`, `pageSize: 50`.
- 계정 유입: `<PRIMARY_EMAIL>` 직접 수신 + `<FORWARDED_EMAILS>` 자동전달분.

## 2. 분류 (응답 필요 여부를 우선 축으로)
- **✍️ 응답·조치 필요**: 사람이 보낸 답장 요구, 수락/거절·확인·기한 조치(예약·환불·보안 로그인·마감 폼 등). 마커: 🔴 결제·계약·금융·기한, 🟡 그 외.
- **📖 읽음만**: 채용 공고, 뉴스레터, 단순 알림.
- **🗑️ 노이즈(건수만)**: 광고·프로모션·Sale·Deal, 본인이 방금 한 행동(2단계 인증·비번 변경�인증번호·커넥터 연결)에 대한 시스템 알림.
- no-reply 발신은 답장 대상이 아니므로 조치(확인/클릭/취소)로 안내.

## 3. 본문 보강
- ✍️/🔴 항목 중 필요 시 `get_thread`로 핵심 1~2문장. 그 외 snippet 한 줄.

## 4. 메일 접근 링크
- `https://mail.google.com/mail/u/0/#inbox/{messageId}`.

## 5. (요청 시에만) 읽음/스팸 처리 — 항상 먼저 물어본다
- 읽음: `unlabel_message`/`unlabel_thread`로 UNREAD 제거. 스팸: `label_thread`로 SPAM.
- 자동 실행에서는 처리하지 않고 "읽음 후보 / 스팸 후보"만 목록 제시.

## 6. 액션아이템 (번호 출력 + 승인 시에만 캘린더 등록)
- 마감·일정 있는 항목을 **번호 매겨** 출력만. 자동 등록 금지.
- "①③ 추가해" 식 지시 시에만 `~~calendar`(create_event)로 등록(all-day, 사용자 시간대, DESCRIPTION에 출처 링크). "2번 삭제해" → delete_event. 미연결 시 `.ics`.

## 7. 출력 형식
```
## 📬 [날짜] 이메일 요약 ([시간])
총 N건 (응답·조치 A / 읽음만 B / 노이즈 Z)

### ✍️ 응답·조치 필요 (A건)
- 🔴 **[보낸이]** — [[제목](링크)]: [핵심]. → 조치: [할 일]. _(HH:MM)_

### 📖 읽음만 (B건)
- 🟡 **[보낸이]** — [[제목](링크)] _(HH:MM)_

### 🗑️ 노이즈 Z건 — 생략

### ⏰ 액션아이템 (번호만 — "①③ 추가해"로 캘린더 등록)
1. [제목] — [마감]
```

## 규칙
- 언어 한국어, 간결·사실 위주. 추측·과장 금지.
- 24시간 내 0건이면 "최근 24시간 내 새 메일이 없습니다." 한 줄.
- 인사말·메타 코멘트 없이 카드 형태로.
